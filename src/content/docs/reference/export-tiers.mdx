---
title: Export Tiers
description: Control which functions are exposed via Transit using export tiers.
---

Transit uses a tiered system to control which functions are exposed via the generated language handles. This allows you to gradually expose functionality as needed.

## Tier 1: Public Functions

By default, Transit scans for publicly accessible functions in each language:

- **Rust**: `pub fn` functions
- **Java**: `public` methods
- **Python**: Top-level `def` functions and public class methods (not starting with `_`)

All such functions are automatically exported as Tier 1 exports.

## Tier 2: File‑Level Annotations

You can further restrict exports using file‑level annotations:

- Add `// transit:file` (Rust/Java) or `# transit:file` (Python) to a source file.
- Only functions in that file that meet the Tier 1 criteria are exported.
- Functions in the file that do not meet Tier 1 criteria are ignored.

## Tier 3: Function‑Level Annotations

For fine‑grained control, annotate individual functions:

- `// transit:function` (Rust/Java) or `# transit:function` (Python)
- Only the annotated function is exported, regardless of its visibility.
- This allows you to export private or internal functions when needed.

## Usage Example

### Rust

```rust
// src/lib.rs

// All pub fn in this file are Tier 1 exports
pub fn public_fn() {}

// This file‑level annotation limits exports to this file only
// transit:file

// This function is exported (Tier 1 because it's pub)
pub fn exported() {}

// This function is NOT exported because it's not pub
fn internal() {}

// This function is exported via Tier 3 annotation
// transit:function
fn private_exported() {}
```

### Java

```java
// src/java/com/example/Service.java

// All public methods in this file are Tier 1 exports
public void publicMethod() {}

// transit:file
public void fileScoped() {}

// Not exported (not public)
void internal() {}

// Exported via Tier 3 annotation
// transit:function
private void privatelyExported() {}
```

### Python

```python
# service.py

# All top-level functions are Tier 1 exports
def public_func():
    pass

# transit:file
def file_scoped():
    pass

def _internal():
    pass

# Exported via Tier 3 annotation
# transit:function
def privately_exposed():
    pass
```

## Configuration

Export tiers are detected automatically by the scanner; no additional configuration is required. The scanner respects the annotations and visibility rules described above.

## Interaction with Language Handles

When you create a language handle via `transit.rust(dir)`, `transit.java(dir)`, or `transit.python(dir)`, the resulting `FunctionProxy` will only expose functions that satisfy the active export tier rules for each source file.
